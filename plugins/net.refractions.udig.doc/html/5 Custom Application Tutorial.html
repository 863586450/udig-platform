So you just picked up uDig, and you made it through the first plug-in tutorial - but a little ruler is not going to make that great a demo for your boss? Do we have a tutorial for you ... in a matter of (several) minuets you can whip up a custom application - with your name on it!

Normally there are two ways to do this:

  • create a new RCP Application, using your own plug-ins and uDig plug-ins
  • drop the uDig plug-ins into an already existing RCP application (if you are in this situation you are an expert go ahead and start hacking, uDig is set up for what you need)

There are lots of tutorials out there on the web about creating Eclipse RCP Applications. The mechanics are going to be exactly the same here, we are just going to be using uDig components for the example.

Packaging your Content

There are two ways to organize an RCP application, as a pile of plugins, or organized into "features". An RCP "feature" is a collection of plug-ins that are designed to work together.

The uDig SDK is comprised of several features:

  • gisplatform
  • gisapplication
  • ...

Let's set up a feature for our distance tool created in the earlier tutorial.

  • create feature: net.refractions.udig.example-feature
  • list distance tool plugin as contents

tip: You can set up features like this as part of the uDig community svn, features are available for others to try out and serve as a great way to get involved with the uDig project.

Creating a Product

The next step is to define a "product" configuration that can be bundled up for execution (and distribution).

  1. create a plugin: net.refractions.udig.example.ui
  2. create a product: net.refractions.udig.example.ui/example.product
  3. open up example.product
  4. add your feature
  5. add the uDig features ...
  6. choose uDig application from the list
  7. Run ....

There we have it your own custom application But wait there is more ...

Creating a Branding Plugin

You probably did not notice, but in the help menu your feature is not sorted into the right spot. By creating a branding plugin we are going to be able to fix that, and also do something you care about - adding a splash screen.

  • create a plugin: net.refractions.udig.example
  • fill in the blanks
  • add a splash screen

Creating an Distrbution

For example.product you can also create a distribution.

  1. open up example.product
  2. hook it up with a custom icon
  3. click the button

(Please not to create a cross platform release you will need to download a few RCP extras)

Where to go from here?

You now know how to create a plugin, bundle it up into a feature, and create an application out of only the features you need.

What is next?

  • An update site is used to publish features for easy download, you can walk through the instructions in the Eclipse help to create an update site for our example.feature.
  • Our example application was rather minimal, why not add in an additional uDig feature like printing? You may also be able to find some interesting community features to try out.
